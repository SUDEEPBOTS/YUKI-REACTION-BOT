## 📋 How to Contribute

### 🐛 Bug Reports

1. Check if the bug is already reported in [Issues](https://github.com/SUDEEPBOTS/Downloader/issues)
2. If not, open a new issue with:
   - **Title**: Clear, descriptive title
   - **Description**: Steps to reproduce the bug
   - **Expected behavior**: What should happen
   - **Actual behavior**: What actually happens
   - **Logs**: Any relevant error logs

### 💡 Feature Requests

1. Open an issue with the `enhancement` label
2. Describe the feature and why it would be useful
3. Include any relevant examples or mockups

### 🔧 Pull Requests

1. **Fork** the repository
2. **Clone** your fork:
   ```bash
   git clone https://github.com/YOUR_USERNAME/Downloader.git
   ```
3. **Create** a feature branch:
   ```bash
   git checkout -b feature/your-feature-name
   ```
4. **Make** your changes and commit:
   ```bash
   git commit -m "✨ Add: your feature description"
   ```
5. **Push** to your fork:
   ```bash
   git push origin feature/your-feature-name
   ```
6. **Open** a Pull Request to the `main` branch

## 📏 Code Style Guidelines

| Rule | Details |
|------|---------|
| **Language** | Python 3.11+ |
| **Framework**| FastAPI |
| **Naming** | `snake_case` for functions, `UPPER_CASE` for constants |
| **Copyright** | Keep the copyright header on all `.py` files |
| **Commits** | Use emoji prefixes: `✨ Add`, `🐛 Fix`, `📝 Docs`, `♻️ Refactor` |

---

## ⚠️ Important Rules

- Do **NOT** commit any tokens, API keys, or cookies
- Do **NOT** remove or modify the copyright headers or anti-tamper logic
- **DO** test your changes before submitting a PR
